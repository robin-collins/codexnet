## Security review

- UniFi POST is possible only when both the method and exact API-specific login path
  match. All other POST/write shapes fail before the transport adapter is called.
- Cookies, CSRF tokens, passwords, helper output, and exception bodies are not persisted
  or emitted. Credential files require safe ownership/mode and helper keys use standard
  input rather than process arguments.
- Controller selection is restricted to configured endpoints; the CLI `--controller`
  option can only narrow that list. Discovery derives candidates from stored evidence
  and performs no credential or endpoint probe.
- AD collection targets pass the shared approved IPv4-range boundary. Detection has no
  credential parameter and rejects out-of-domain names and out-of-range addresses before
  RootDSE access.
- Repository/dependency searches found no Impacket, BloodHound, secretsdump, Kerberoast,
  AS-REP roasting, DCSync, password spraying, or brute-force implementation/dependency.
- Trusts and memberships are documentation records only; the report does not infer or
  present attack paths.
