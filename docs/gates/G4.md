# G4 infrastructure collector gate

Verified on 2026-07-15 at commit `d19ad64` using sanitized fixtures, mocks, disposable
loopback-only protocol emulators, temporary directories, and read-only protected-state checks. No
customer device or non-loopback address was contacted, and no system service was changed.

## Quality and collector evidence

- `ruff format --check` and `ruff check` passed across 63 files; strict `mypy src` passed across 31
  source files.
- Pytest passed 871 tests with one optional LibreOffice test skipped. Line and branch coverage were
  both 100%.
- The focused G4 matrix passed 177 collector-framework, SNMP, SSH, infrastructure-reporting,
  loopback-transport, and CLI tests. It covers success, timeout, authentication rejection, partial
  results, cancellation, bounded retries/concurrency, unexpected failure isolation, and one
  collector failing while independent collectors complete.
- Requests outside configured IPv4 ranges are rejected before a run or socket operation. CLI and
  orchestrator tests cover invalid, IPv6, overly broad, and unapproved targets.
- A clean Pi-side virtual environment installed the exact lock successfully. `pip check` passed;
  the live SSH stack resolved to Netmiko 4.7.0, NTC Templates 9.2.0, Paramiko 4.0.0, and TextFSM
  2.1.0, with ARM64 compatibility documented in `docs/dependencies.md`.

## SNMP evidence

- SNMPv3 is the default and rejects missing/invalid profiles without echoing values. SNMPv2c is
  accepted only with both an explicit v2c profile and `allow_insecure_v2c: true`; no default or
  guessed community exists, and the CLI emits a security notice.
- Mocked transport tests cover v3/v2c authentication construction, GET and bounded walks, timeout,
  authentication/authorization failure, unsupported OIDs, partial responses, cancellation, global
  table limits, and bounded unknown-OID retention.
- Sanitized Cisco-compatible, Aruba-compatible, printer, UPS, and environment fixtures normalize
  interfaces, addresses, LLDP, bridge MACs, neighbors, VLANs, PoE, sensors, consumables, counters,
  and firmware while preserving source, timestamp, native units, unknown values, and partial data.
- The committed lab test crosses the real PySNMP UDP boundary to a disposable agent bound to an
  unprivileged `127.0.0.1` port. It verifies a real response, timeout, configured port bounds, v2c
  opt-in, and redaction before SQLite persistence.
- A supplemental isolated user/network-namespace run crossed the real authPriv SNMPv3 boundary to
  a disposable local Net-SNMP agent on `127.0.0.1:161`. It normalized 11 facts including `sysName`;
  the synthetic authentication and privacy values were absent from SQLite. The same setup verified
  explicit v2c behavior and pre-collection refusal of an out-of-scope address.

## SSH command and transport evidence

- Fixture sessions cover Cisco IOS, HP/HPE Comware, and ArubaOS-Switch families, conservative
  platform selection, paging, authentication/timeout failure, parser fallback, partial and
  oversized output, cancellation, credential-provider restrictions, and bounded sanitized raw
  artifacts.
- Every application and pinned-driver session command is in an exact reviewed allowlist.
  Abbreviations, extra arguments, `configure terminal`, configuration display, copy/save/write,
  reload, and arbitrary commands are rejected. The loopback emulator independently rejects any
  command outside that allowlist.
- The real loopback lab test crosses Netmiko and Paramiko over TCP to a disposable synthetic device
  on an unprivileged `127.0.0.1` port. It uses a generated host key pinned in a temporary known-hosts
  file with strict verification, an explicit read-only profile, and a synthetic credential. All
  operational commands completed, no write/configuration command was observed, and credentials
  were absent from SQLite and artifacts.
- Netmiko's Aruba/ProCurve drivers enter enable mode during automatic setup. CodexNet therefore
  uses the generic SSH channel for Aruba and sends only its own audited commands; output may be an
  honestly disclosed parse fallback. Cisco and Comware setup remains constrained to audited
  session display controls and logout.
