# G3 passive discovery and topology gate

Verified on 2026-07-15 using sanitized fixtures, temporary directories, staged packaging, and
read-only host-state checks. Per the audit instruction, no live capture or system installation was
performed.

## Quality and replay evidence

- `ruff format --check`, `ruff check`, and strict `mypy src` passed.
- Pytest: 568 passed, 1 skipped; line and branch coverage were both 100%.
- An accelerated 24-hour timestamp replay exercised production LLDP/CDP, mDNS, DHCP, ARP, and
  kernel-neighbour paths. It processed 144 frames, isolated all 24 injected malformed frames, and
  retained no payload attributes.
- The production queue peaked at 6 of 256 entries. Its maximum transient queued payload capacity
  was 2,359,296 bytes; `tracemalloc` peak for the controlled replay was 113,720 bytes. Stateful
  parser caches and kernel output/record counts are independently bounded by tests.
- Near the replay horizon, topology contained 9 active edges covering link-layer neighbours,
  service advertisement/hosting, DHCP, VLAN, and subnet membership. A changed kernel mapping
  remained an explicit `neighbor_ip_reuse` conflict.
- At the later expiry snapshot, 309 expired evidence items were excluded and no stale edge was
  rendered. Reversing input order produced identical diagram source and digest.
- The bounded kernel command tests cover success, non-zero exit, timeout, oversize output,
  cancellation, malformed JSON, record limits, and fixed-size subprocess cleanup. Silent 24-hour
  expiry and mocked service restart/history tests pass.

## Service and packaging evidence

- `systemd-analyze verify` passed against a staged unit with only `ExecStart` replaced by
  `/bin/true`; the repository test performs the same parse check.
- `systemd-analyze security --offline=yes` rated the unit 1.2 (OK).
- The unit runs as `field-discovery`, grants only `CAP_NET_RAW`, allows only `AF_UNIX`,
  `AF_PACKET`, and the kernel-neighbour query's `AF_NETLINK`, blocks Internet socket families, and
  applies restart, rate, CPU, memory, task, file, syscall, and writable-path limits.
- Staged install/remove created and removed only the CodexNet unit and sysusers files. It did not
  enable or start a service and retained application/configuration/data paths.
- Reboot/restart behavior was verified statically through `WantedBy=multi-user.target`,
  `Restart=on-failure`, start limits, staged enable/install semantics, and mocked runtime restart.
